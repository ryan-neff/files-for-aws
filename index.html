Vapor: Exploring Server Side Scripting with Swift

For this exploration I wanted to explore one of the newest languages in the field; Swift. Swift has been out for a couple years now and only just recently has it become open source. This means many new and interesting projects are being implemented for the language outside of iOS development. This includes using Swift as a server-side language. Swift has been praised for its speed over many other languages, which is a benefit when you have a web application sending and requesting several packets over the wire.

Vapor is the new attempt at a web framework for Swift which was only released officially a couple of days ago. Already, in a speed test to see how many JSON requests it can process in a second, Vapor comes in third in front of several popular frameworks used today.

So I decided “Why not take a stab at it?”. To preface, I have very little experience with Swift and any experience I had was too long ago to really even remember. Also, since this framework is so new, there is basically no help on StackOverflow what-so-ever. PLUS any tutorials online where written when Vapor was in beta, so much has changed in the framework syntactically from beta to release that those also weren’t much help. Not to mention dependency issues as well. SO I was sort of on my own…. ANYWAYS rant over.

Setting up Vapor could not have been easier, the only requirements are to have Xcode 8 installed and the latest version of Swift (Swift 3). After that you just type in vapor new [project_name] and Vapor will set everything up for you. The folder structure is a standard MVC set-up (Controllers in the Controllers folder, Models in the Models, etc.). The main piece of the framework is a Class called the Droplet. The Documentation states:

"The Droplet is a service container that gives you access to many of Vapor's facilities. It is responsible for registering routes, starting the server, appending middleware, and more."

You can think of this class mainly as the conductor, routing you to the correct place, retrieving data, etc. Take this Hello World example: 

		drop.get("hello") { request in
    			return "Welcome"
		}

Here when the url “project.com/hello” is called, the String “Welcome” will be returned to your screen.

Once I began getting my feet wet playing with this Droplet. I wrote a couple functions that were similar in nature. They were:

  • Output a JSON Object to the User
  • Retrieve A JSON Document from the Web (wikipedia) and output it to the user
  • Rout the user to make google search on their name
  • Re-rout the user to a different view

All in all it was a great first experience with Vapor. As the project sits now, there is a bit of a steep learning curve if you are coming to it with little or no experience with Swift (go figure). But as the popularity of this language keeps increasing, it is not unlikely that you will see more applications being developed with this framework. Additionally, the existence of a project like this only motivates me more to learn and work with Swift more often.
